ProductProxy App Documentation

This App gives to the RoadCube POS the products selected by the cashier. The first step is to synchronize the products at the cashier with RoadCube Platform. This can be done with User Interface (point1) or via web service (point2). Then with the point 3 we can send in real time the products to the RoadCube POS. Point 4 is used only by RoadCube POS.
1) Synschronizing Products (User Interface)

URL:
http://roadcu.be/ProductsProxy/syncProductsUI.php


You can easily upload the products by giving the JSON in the textarea and click submit. Products are synced with RoadCube.


2) Synschronizing Products (REST Web Service)

URL:
http://roadcu.be/ProductsProxy/syncProducts.php?storeID=XXXXX&token=YYYYYY&products={"ProductID1":Price1,"ProductID2":Price2}

"products" param example format: {"323":23.2,"442":99.09} for 2 products with ID 232(price $23.2) and ID 442(price $99.09)

Important* for the prices use dot(.) not comma(,)

SUCCESS response:
SUCCESS


FAIL response:
FAIL
3) Send Product Request (REST Web Service)

This web service is used by cashier machines to send the products selected.

URL:
https://roadcu.be/ProductsProxy/REQUEST.php?storeID=XXXXX&token=YYYYYY&products=Product1,Product2


SUCCESS response:
SUCCESS


FAIL response:
FAIL

4) Get Product Request (REST Web Service)

This web service is read by the RoadCube POS when products are needed to be obtained during a transaction process.

URL:
https://roadcu.be/ProductsProxy/checkRequests.php?storeID=XXXXX


SUCCESS response:
Product1,Product2,Product3


FAIL response:
NULL
